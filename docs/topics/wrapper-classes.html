Interview brief + practice drills

Good to knowTrap questions

Integer cache (-128 to 127) and == vs equals on wrappers are classic trick questions. Know them before any interview.

Interviewer signals
  • Integer cache range?
  • == vs equals on Integer?
  • Autoboxing pitfalls?

Quick recall sheet

Memorize these short answers — expand with code examples in each lesson.

Why wrappers?
Collections and generics only work with objects, not primitives.
Autoboxing
Automatic conversion from primitive to wrapper, e.g. `int` -> `Integer`.
Unboxing
Automatic conversion from wrapper to primitive, e.g. `Integer` -> `int`.
`Integer a = 127; Integer b = 127; a == b`
`true` due to Integer cache (`-128` to `127`).
`Integer c = 128; Integer d = 128; c == d`
Usually `false` (different objects).
Wrapper comparison best practice
Use `equals()` for value comparison, not `==`.
Risk with unboxing
`Integer x = null; int y = x;` throws `NullPointerException`.
`parseInt` vs `valueOf`
`parseInt` returns primitive `int`; `valueOf` returns `Integer`.

Quick Notes

  • Prefer primitives for heavy numeric computation when null is not needed.
  • Use wrappers when APIs require objects (List<Integer>, Map<Long, String>, etc.).
  • Be careful with autoboxing in tight loops due to object creation overhead.
