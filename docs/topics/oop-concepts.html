Chapter: OOP Concepts

Interview brief + practice drills

Must-knowEvery round

Interviewers probe OOP to see if you understand design — not just syntax. Expect equals/hashCode, overriding rules, and abstract class vs interface.

Interviewer signals
  • Can you explain runtime polymorphism?
  • Why override hashCode with equals?
  • Abstract class vs interface — when which?

Quick recall sheet

Memorize these short answers — expand with code examples in each lesson.

Abstraction vs Encapsulation
Abstraction hides *what* (design); Encapsulation hides *how* (implementation/data)
Abstract class vs Interface
Abstract class has state + partial impl; Interface is a pure contract (Java 8+ allows default methods)
Overloading vs Overriding
Overloading = same name, diff params (compile-time); Overriding = same sig in subclass (runtime)
Why override `hashCode` with `equals`?
Contract: equal objects must have same hash. Breaks HashMap/Set otherwise
`IS-A` vs `HAS-A`
IS-A = Inheritance; HAS-A = Composition/Aggregation
Can constructors be inherited?
No. But `super()` calls parent constructor
Can we override `static` methods?
No — static methods are hidden, not overridden (no runtime polymorphism)
Can we override `private` methods?
No — private methods are not visible to subclass
